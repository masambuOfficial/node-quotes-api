Quotes API Documentation

Introduction

The Quotes API is a RESTful web service that allows you to manage a collection of quotes. Each quote has a unique serialized ID (UUID) rather than an incremental integer ID, ensuring that each quote can be uniquely identified across different systems and contexts. The API supports basic CRUD (Create, Read, Update, Delete) operations for quotes.

Base URL

http://localhost:4500

Endpoints Overview

  • GET /quotes: Retrieve all quotes.
  • GET /quotes/:id: Retrieve a single quote by its ID.
  • POST /quotes: Create a new quote.
  • PUT /quotes/:id: Update an existing quote by its ID.
  • DELETE /quotes/:id: Delete a quote by its ID.
  • GET /authors: Retrieve all authors.
  • GET /authors/:id: Retrieve a single author by their ID.
  • POST /authors: Create a new author.
  • PUT /authors/:id: Update an existing author by their ID.
  • DELETE /authors/:id: Delete an author by their ID.

Data Format

Quotes and authors are stored as JSON objects in the following format:

[
    [
        {
            "id": 2,
            "text": "\"Success is not final, failure is not fatal: It is the courage to continue that counts.\"",
            "year": 1942,
            "category": "Motivational",
            "authorId": 3
        },
        {
            "id": 8,
            "text": "\"Be the change you wish to see in the world.\"",
            "year": 1913,
            "category": "Motivation",
            "authorId": 11
        },
    ]
]
  • `id`: A auto-incremented ID is generated as data is being saved in the DB.
  • `text`: The text of the quote.
  • `year` (optional): The year the quote was said or published.
  • `category`: The category of the quote. e.g motivation, love e.t.c
  • `authorID`: This a Foreign Key (FK) creating a relationship between quotes and authors

Authors are stored as JSON objects in the following format:

[
    {
        "id": "1",
        "name": "Franklin D. Roosevelt",
        "picture": "http://example.com/image.jpg"
    },
    {
        "id": "2",
        "name": "Albert Einstein",
        "picture": "http://example.com/image2.jpg"
    }
]
  • `id`: A auto-incremented ID is generated as data is being saved in the DB.
  • `name`: The name of the author.
  • `picture` (optional): The URL to the author's picture.

API Endpoints

1. Retrieve All Quotes

Endpoint: GET /quotes

Description: Retrieves all quotes stored in the API.

Response:

  • Status: 200 OK
  • Body: An array of quote objects.

Example Request:

GET http://localhost:4500/quotes

Example Response:

[
                    {
                        "id": 2,
                        "text": "\"Success is not final, failure is not fatal: It is the courage to continue that counts.\"",
                        "year": 1942,
                        "category": "Motivational",
                        "authorId": 3
                    },
                    {
                        "id": 8,
                        "text": "\"Be the change you wish to see in the world.\"",
                        "year": 1913,
                        "category": "Motivation",
                        "authorId": 11
                    },
]

2. Retrieve a Single Quote by ID

Endpoint: GET /quotes/:id

Description: Retrieves a specific quote by its unique serialized ID.

Response:

  • Status: 200 OK
  • Body: A quote object.

Example Request:

GET http://localhost:4500/quotes/1

Example Response:

{
                    "id": 2,
                    "text": "\"Success is not final, failure is not fatal: It is the courage to continue that counts.\"",
                    "year": 1942,
                    "category": "Motivational",
                    "authorId": 3
}

3. Create a New Quote

Endpoint: POST /quotes

Description: Creates a new quote entry.

Request Body:

{
                    "text": "Because I could not stop for Death, he kindly stopped for me; the carriage held but just ourselves and Immortality." ,
                    "year": 1890,
                    "category": "Motivation",
                    "name": "Emily Dickinson",
                    "picture": "https://rb.gy/pzo6fd"
}

Response:

  • Status: 201 Created
  • Body: The created quote object, including its generated ID.

Example Request:

POST http://localhost:4500/quotes
Content-Type: application/json

{
    "text": "Because I could not stop for Death, he kindly stopped for me; the carriage held but just ourselves and Immortality." ,
    "year": 1890,
    "category": "Motivation",
    "name": "Emily Dickinson",
    "picture": "https://rb.gy/pzo6fd"
}

Example Response:

{
                    "id": 7,
                    "text": "Because I could not stop for Death, he kindly stopped for me; the carriage held but just ourselves and Immortality.",
                    "year": 1890,
                    "category": "Motivation",
                    "authorId": 8,
                    "Author": {
                        "id": 8,
                        "name": "Emily Dickinson",
                        "picture": "https://rb.gy/pzo6fd"
                    }
}

4. Update a Quote by ID

Endpoint: PUT /quotes/:id

Description: Updates an existing quote by its unique ID.

Request Body:

{
    "quote": "The only limit to our realization of tomorrow is our doubts of today.",
    "author": "Franklin D. Roosevelt",
    "year": 1945
}

Response:

  • Status: 200 OK
  • Body: The updated quote object.

Example Request:

PUT http://localhost:4500/quotes/1
Content-Type: application/json

{
    "quote": "The only limit to our realization of tomorrow is our doubts of today.",
    "author": "Franklin D. Roosevelt",
    "year": 1945
}

Example Response:

{
    "id": "1a2b3c4d-5e6f-7g8h-9i0j-k1l2m3n4o5p6",
    "quote": "The only limit to our realization of tomorrow is our doubts of today.",
    "author": "Franklin D. Roosevelt",
    "year": 1945
}

5. Delete a Quote by ID

Endpoint: DELETE /quotes/:id

Description: Deletes a specific quote by its unique serialized ID.

Response:

  • Status: 204 No Content

Example Request:

DELETE http://localhost:4500/quotes/1

How to Use

The API uses a PostgreSQL database with Prisma for ORM. Ensure your database is running on port 5432 (or as configured) and your Prisma schema is properly set up.

  • Make sure to have PostgreSQL and Prisma installed and configured.
  • Run your server on port 4500.
  • Use Postman or a similar tool to test the API endpoints.

Errors

Common error responses include:

  • 400 Bad Request: Invalid request data.
  • 404 Not Found: The requested resource could not be found.
  • 500 Internal Server Error: An error occurred on the server.
